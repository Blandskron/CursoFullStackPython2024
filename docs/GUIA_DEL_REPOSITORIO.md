# Guía del Repositorio Sustantiva

Documento de referencia para docentes y estudiantes que necesiten entender rápidamente el alcance académico y la organización interna del proyecto.

## 1. Objetivo pedagógico

Sustantiva consolida ejercicios prácticos para un itinerario formativo en desarrollo web y programación, pasando por:

- Fundamentos de interfaz (Frontend).
- Programación con Python (niveles base e intermedio).
- Modelado y consulta de datos con PostgreSQL.
- Introducción al desarrollo web con Django.

## 2. Mapa de módulos

## 2.1 `02Frontend`

Contiene prácticas de:

- HTML y maquetación inicial.
- CSS (estilos, layouts, resets).
- JavaScript en navegador.
- jQuery y Bootstrap.
- Actividades guiadas por día (`SemanaX/DiaY`).

Perfil de uso:

- Clases de introducción al desarrollo web.
- Laboratorios de UI/UX básicos.
- Talleres de interacción DOM.

## 2.2 `03Python`

Enfocado en fundamentos de programación:

- Tipos de datos y estructuras.
- Condicionales, ciclos y funciones.
- Ejercicios progresivos por lección.
- Casos de uso prácticos y notebooks.

Perfil de uso:

- Nivel inicial-intermedio.
- Refuerzo de pensamiento computacional.

## 2.3 `04Python2`

Extiende la práctica hacia nivel intermedio/avanzado:

- Algoritmos clásicos (ordenamiento, búsqueda, recursión).
- Estructuras de datos.
- Retos complementarios (`BonusPythonV1/V2/V3`).
- Colección adicional de notebooks de apoyo.

Perfil de uso:

- Entrenamiento técnico.
- Preparación para entrevistas junior.
- Actividades de profundización.

## 2.4 `05Basededatos`

Incluye material para SQL y PostgreSQL:

- Consultas básicas y avanzadas.
- Manipulación e importación de datos.
- Funciones y unión de tablas.
- Scripts de apoyo para clases prácticas.

Perfil de uso:

- Talleres de modelado relacional.
- Laboratorios de consultas y reportes.

## 2.5 `06DjangoV1`

Bloque de introducción al framework Django:

- Contexto inicial del framework.
- Creación de proyectos.
- Despliegue de páginas dinámicas.

Perfil de uso:

- Cierre de ruta full-stack inicial.
- Conexión de frontend + backend + datos.

## 3. Convenciones de organización

- Estructura jerárquica por `Módulo > Semana > Día`.
- Nomenclatura mayormente descriptiva orientada al contenido de clase.
- Mezcla de formatos prácticos: `.py`, `.sql`, `.html`, `.ipynb`, assets multimedia.

## 4. Recomendaciones para mantenimiento docente

1. Agregar un `README.md` por módulo con objetivos semanales.
2. Estandarizar nombres de archivos para búsquedas más rápidas.
3. Definir rúbricas por actividad (entrada, proceso, salida esperada).
4. Incorporar solucionarios separados cuando corresponda.
5. Mantener un índice de ejercicios destacados por nivel.

## 5. Sugerencias para estudiantes

- Resolver los ejercicios sin mirar soluciones en primera instancia.
- Reescribir scripts clave con mejoras de estilo y modularidad.
- Construir portafolio personal a partir de proyectos de cada bloque.
- Documentar aprendizajes, errores y mejoras en un cuaderno técnico.

## 6. Uso recomendado en programas formativos

- **Bootcamps**: como ruta troncal de práctica diaria.
- **Cursos universitarios/técnicos**: como banco de laboratorios.
- **Autoaprendizaje**: como plan escalonado con objetivos semanales.

## 7. Próximos pasos sugeridos para profesionalización

- Añadir licenciamiento explícito.
- Integrar plantillas de issues y pull requests.
- Crear checklists de evaluación por módulo.
- Implementar validaciones básicas automáticas (lint/test) para Python y SQL.
